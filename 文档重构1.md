## 一、程序怎么跑的

### 启动链路

```
index.html
  └─ src/main.tsx                    # React 入口
       └─ src/App.tsx                # 根组件，组合所有逻辑
            ├─ usePlayerStore()      # 从 electron-store 读取全部状态
            ├─ useAudioPlayer()      # 绑定 HTMLAudioElement 事件
            └─ <FloatingPlayer />    # UI 入口
                 └─ <ExpandedPanel />
                      ├─ <Playlist />
                      ├─ <FavoritesTab />
                      └─ <RecentTab />
```

### 关键数据流

```
用户操作 → App.tsx handler → api.ts (IPC invoke) → electron/main.ts (fetch Bilibili)
                                    ↓
                             usePlayerStore → electron-store (持久化)
                                    ↓
                             useAudioPlayer → HTMLAudioElement (播放)
```

### 三个入口点

| 入口 | 触发方式 | 作用 |
|------|----------|------|
| 输入 BV 号/链接 | ExpandedPanel 输入框 → `handleInputSubmit` | 加载视频/收藏夹到播放列表 |
| 点击播放按钮 | 悬浮球/ExpandedPanel → `handlePlayPause` | 播放/暂停 |
| 点击曲目 | Playlist/FavoritesTab/RecentTab → `handlePlayTrack` | 切歌播放 |

## 四、全局变量 & 模块级单例

### ⚠️ 隐式状态（最容易出问题）

| 位置 | 变量 | 类型 | 问题 |
|------|------|------|------|
| `api.ts:17` | `audioEl` | `HTMLAudioElement \| null` | 全局单例，所有播放操作共享 |
| `api.ts:19` | `currentUrl` | `string` | URL 缓存 |
| `api.ts:20` | `currentExpiresAt` | `number` | 过期时间 |
| `api.ts:21` | `currentBvid` | `string` | 当前 bvid |
| `api.ts:22` | `currentCid` | `number` | 当前 cid |

**风险**：模块级变量，不受 React 状态管理。多组件同时操作 `audioEl` 会产生竞争条件。

### React 状态（受控）

| Hook | 状态 | 存储 |
|------|------|------|
| `usePlayerStore` | tracks, currentIndex, playMode, volume, favorites, recentTracks, windowPosition, windowSize, loading | electron-store |
| `useAudioPlayer` | isPlaying, currentTime, duration, volume, currentAudio | useState |
| `FloatingPlayer` | collapsedState | useState |
| `ExpandedPanel` | inputValue, activeTab, seekingValue | useState |

## 七、重构方案

### P0：删除废弃文件

```
删除 src/utils/format.ts      ← 未被引用
删除 src/utils/track.ts       ← 未被引用
删除 src/styles/tokens.css    ← 已被 design-tokens.css 替代
删除 vite.config.electron.ts  ← 未被任何 script 引用
```

**收益**：减少认知负担，消除"这个文件到底在不在用"的困惑。

### P1：拆分 App.tsx（257 行 → 3 个 hooks）

当前 App.tsx 有 15+ 个 handler，职责过重。

```
src/hooks/
  ├── usePlayerStore.ts        # 已有
  ├── useAudioPlayer.ts        # 已有
  ├── usePlaylistActions.ts    # 新增：handlePlayTrack, handleDeleteTrack, handleReorderTracks, handleClearPlaylist
  ├── useFavoriteActions.ts    # 新增：handleCreateFavorite, handleToggleFavorite, handleAddToFavorite, ...
  └── useInputHandler.ts       # 新增：handleInputSubmit, handleAddToFavoriteFromInput
```

**收益**：每个 hook 职责单一，可独立测试，改一处不影响其他。

### P2：拆分 FloatingPlayer.tsx（314 行 → 2 个 hooks）

当前拖拽、缩放、折叠逻辑混在一起，是 bug 高发区。

```
src/hooks/
  ├── useWindowDrag.ts    # 新增：mousedown/mousemove/mouseup 拖拽逻辑
  └── useWindowResize.ts  # 新增：e/se/s 三向缩放逻辑
```

**收益**：窗口行为可独立调试，拖拽 bug 不影响缩放，反之亦然。

### P3：封装 api.ts 全局单例

当前 5 个模块级变量不受 React 管理。

```
方案 A：封装为 AudioPlayer 类
  class AudioPlayer {
    private audioEl: HTMLAudioElement
    private urlCache: { url, expiresAt, bvid, cid }
    play(track): Promise<void>
    pause(): void
    seek(time): void
    volume(v): void
    refreshUrl(): void
  }

方案 B：用 React Context 管理（改动更大）
```

**建议**：方案 A 改动最小，风险最低。

### 优先级排序

| 优先级 | 改动 | 风险 | 收益 |
|--------|------|------|------|
| P0 | 删除废弃文件 | 无 | 消除困惑 |
| P1 | 拆分 App.tsx | 低 | 降低胶水层复杂度 |
| P2 | 拆分 FloatingPlayer | 中 | 减少窗口 bug |
| P3 | 封装 api.ts | 中 | 消除隐式状态 |
