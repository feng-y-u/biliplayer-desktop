# .harness — 轻量工程工作台

约束 AI 开发行为。解决四大问题：上下文失控、提前完成、链路不通、假绿测试。

**核心规则 12 条，见 `agent-owner.md`。**

## 工作流程

```
规划 → 编码 → 验证 → 记录
 │       │       │       │
 │       │       │       └─ changes/  记录变更
 │       │       └─ 验证清单 + tsc --noEmit + 手动测试
 │       └─ rules/ 约束编码行为
 └─ templates/ 需求拆解模板
```

### 每次变更的标准流程

1. **规划**：在 `changes/<date>-<topic>/` 创建目录，填写需求拆解（3~5 条子任务）
2. **编码**：遵循 12 条核心规则
3. **验证**：tsc --noEmit + 手动测试核心流程，附验证结果
4. **记录**：更新 changes/ 记录，报告完成时附验证截图/日志

## 目录索引

| 目录 | 用途 |
|------|------|
| `agent-owner.md` | 总纲：12 条核心规则 + 架构要点 |

| `skills/` | 工作技能指南 |
| `references/` | 参考资料索引 |
