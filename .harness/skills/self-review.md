# 自检评审

对应规则 #5-7：测试规则（防假绿）。

## 检查清单

### 规则 5：真链路验证

- [ ] 核心流程跑过真实环境（不是 mock）
- [ ] 人工确认跑过

### 规则 6：相关测试

- [ ] `npx tsc --noEmit` 通过
- [ ] 手动测试修改涉及的功能
- [ ] CI 通过（如有）

### 规则 7：业务逻辑确认

- [ ] 先读代码，理解业务逻辑
- [ ] 再看测试，确认覆盖关键路径
- [ ] 最后看测试结果

## 常见假绿场景

- 测试通过但实际功能不工作
- 测试覆盖了代码但没覆盖业务逻辑
- 测试用例本身有 bug
- 依赖的外部服务返回了非预期数据
