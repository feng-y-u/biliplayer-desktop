# 边界规则（防乱改）

### 规则 1：只改当前任务涉及的文件

**描述**：不顺手改无关代码。只动任务明确要求改的文件。

**验证方式**：人工看 diff，确认没有意外修改。

**具体要求**：
- 不改周围代码的格式/命名（除非任务明确要求）
- 不顺手重构没提到的模块
- 不加任务无关的注释
- 不删任务无关的代码

**反例**：
```
任务：修复播放按钮样式
diff 里却改了 Playlist.tsx 的排序逻辑  ← 违规
```

### 规则 2：新增文件/模块要说明

**描述**：新增文件或模块时，在 `agent-owner.md` 的"架构要点"部分简单说明。

**验证方式**：检查 `agent-owner.md` 是否有对应说明。

**格式**：
```markdown
- **新模块名**：简要说明用途
```
